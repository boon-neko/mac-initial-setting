# Research Skill - 最新情報調査

## 概要

技術トピックについて最新情報を調査し、Gemini CLIを活用して壁打ちしながら深掘りするスキル。

## 起動方法

```
/research <調査トピック>
```

## Prerequisites

CLAUDE.md の「Workflow Conventions」から **Research Output** ディレクトリを読み取る。
未定義の場合はデフォルト `docs/research/` を使用。

## プロセス

### Phase 1: 初期調査（WebSearch）

1. WebSearchで最新情報を収集
2. 関連するGitHubリポジトリ、Hugging Face、公式ドキュメントを探す
3. ベンチマーク結果や比較記事を探す

### Phase 2: Gemini壁打ち

Gemini CLIを使って調査内容を深掘り:

```bash
# 調査内容についてGeminiに質問
gemini -p "以下のトピックについて、私が見落としている観点や追加で調べるべきことを教えて: <トピック要約>"

# 技術的な詳細を確認
gemini -p "以下の技術選択について、メリット・デメリットを整理して: <選択肢一覧>"

# 実装アドバイス
gemini -p "以下の構成で実装する場合の注意点は？: <構成案>"
```

壁打ちのコツ:
- 1回の質問は1つのトピックに絞る
- Geminiの回答で新しい観点が出たら、WebSearchで追加調査
- 3-5回の壁打ちで十分な深さになる

### Phase 3: レポート作成

調査結果を以下の形式でまとめる:

```markdown
# 調査レポート: <トピック>

## 調査日
YYYY-MM-DD

## サマリー
- 結論を3行以内で

## 詳細

### 現状
- 現在の状況、利用可能な選択肢

### 比較表
| 項目 | 選択肢A | 選択肢B | 選択肢C |
|------|---------|---------|---------|

### メリット・デメリット
- 各選択肢の長所短所

### 推奨
- 推奨する選択と理由

### 注意点
- 実装時の注意点、ハマりポイント

## 参考リンク
- [リンク1](URL)

## Gemini壁打ちメモ
- 壁打ちで得られた追加の洞察
```

### Phase 4: 保存

レポートを CLAUDE.md の Research Output ディレクトリに保存:
```
{RESEARCH_OUTPUT_DIR}/YYYY-MM-DD-<topic>.md
```

## 重要な原則

- **最新情報優先**: 調査時点での最新情報を必ず確認（日付を明記）
- **一次情報重視**: 公式ドキュメント、GitHubリポジトリを優先
- **実用性重視**: 理論より実際に動くか、使えるかを重視
- **壁打ち活用**: Geminiで見落としがないか確認
- **ソース明記**: 全ての情報にソースを付ける
